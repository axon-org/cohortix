## 🔑 The #1 New Rule: Requirements Discovery Protocol

**BEFORE writing any spec or code, ASK THESE 5 QUESTIONS:**

### 1. Intent & Goals

- What problem does this solve?
- Who is this for?
- What does success look like?

### 2. Scope & Boundaries

- What's IN scope?
- What's OUT of scope?
- MVP vs full vision?

### 3. Technical Constraints

- Tech requirements?
- Integrations?
- Performance needs?

### 4. Edge Cases & Errors

- What happens when X fails?
- Known gotchas?

### 5. Acceptance Criteria

- How do we know it's done?
- Who approves it?

**Why:** Asking 5 questions (3 min) prevents building the wrong thing (2+ hours
wasted).

## 🚨 Common Mistakes to Avoid

❌ **"I'll just quickly build this"** → Always do Discovery first  
❌ **"I don't need a spec for this small change"** → Every change needs a spec  
❌ **"I'll write the spec after I code"** → Spec comes BEFORE code  
❌ **"This is obvious, I don't need to ask"** → Ask anyway, it's rarely obvious
